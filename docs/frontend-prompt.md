
# EasyClaw Frontend — Build Prompt

## 0 | Mission

You are a senior frontend engineer. Build and maintain the complete
frontend for **EasyClaw** — a voice-interview-powered OpenClaw setup
guide generator.

- Repo: `https://github.com/Jamessfks/easycalw`
- Frontend: `/frontend` (Vite + React SPA)
- Backend: `/backend` (FastAPI — already built, do not modify)

The product flow is linear:
**Landing → Voice Interview → Guide Generation → Output Display**

Users speak to an AI interviewer about their business, then receive
a personalized OpenClaw setup guide, reference docs, and
initialization prompts — all generated by a Claude Agent SDK pipeline.

## 4 | Design System

### 4.1 Color Palette

```
Surface layers (dark → lighter):
  surface-0: #0a0e17    ← page background
  surface-1: #0f1420    ← cards, modals
  surface-2: #151b2b    ← elevated surfaces
  surface-3: #1c2438    ← hover states

Accent colors:
  cyan:  #22d3ee    ← primary accent, links, focus rings
  blue:  #3b82f6    ← secondary, gradient endpoints
  green: #34d399    ← success states, progress
  amber: #fbbf24    ← warnings, highlights
  rose:  #fb7185    ← errors, destructive actions

Text:
  primary:  #e2e8f0    ← body text
  muted:    #94a3b8    ← secondary text
  dim:      #64748b    ← labels, timestamps
```

## 5 | Routes & Pages

### 5.1 Landing Page — `/` (`EasyClawLanding.jsx`)

The entry point. Must accomplish three things:
1. Explain the product in under 5 seconds of reading
2. Let users start immediately (interview or demo)
3. Show proof of quality (demo outputs, recent guides)

**Sections in order:**

1. **Hero** — Headline + subheadline + two CTAs
   - "Start Interview" (primary, launches Vapi call)
   - "Try a Demo" (ghost, scrolls to demo navigator)
   - Backend health indicator (green dot when API is warm)

2. **Industry Quick-Start** — 6-card grid for fast context:
   - Restaurant/Cafe 🍽️ | Healthcare/Dental 🏥 | Real Estate 🏠
   - Developer/DevOps 💻 | Freelancer/Consultant 📋 | Something Else 🎯
   - Selecting one pre-fills interview context

3. **How It Works** — 3-step horizontal flow:
   - Step 1: Talk to our AI interviewer (2 min)
   - Step 2: We generate your personalized guide (~5 min)
   - Step 3: Follow the guide, paste the prompts, go live

4. **Feature Highlights** — 4 key value props:
   - 12 personalized setup steps
   - 8+ skill recommendations
   - 5 ready-to-paste system prompts
   - Security hardening checklist

5. **Demo Output Showcase** — `DemoNavigator` carousel:
   - 5 demo categories (Restaurant, DevOps, Finance, Content, Healthcare)
   - Click to load a pre-generated guide instantly
   - Each card: icon + title + accent color

6. **Recent Guides** — LocalStorage history strip
   - Shows previously generated guides
   - Resume bar for interrupted interviews (crash recovery)

### 5.2 Live Interview — `/interview-flow` (`InterviewView.jsx`)

Split-panel layout:
- **Left:** `AgentPresence` — AI avatar with 4 states:
  - `idle` — listening avatar, subtle float animation
  - `user-speaking` — waveform bars active
  - `agent-thinking` — thinking avatar, pulse animation
  - `agent-speaking` — talking avatar, ring pulse
- **Right:** `Transcript` — live scrolling messages
  - Role badges: U (user) / A (agent)
  - Timestamps in mono
  - Partial text handling for streaming

**Controls:**
- Mute/unmute toggle
- End interview button (requires min 400 chars / 50 words)
- Talking points coaching card (6 suggested topics)

**State machine** (via `useVapi` hook):
`idle → connecting → active → ended`

On call end: transcript formatted via `POST /format`, then routed
to guide generation.

### 5.3 Demo Interview — `/demo-interview` (`DemoInterviewView.jsx`)

No microphone required. Pre-scripted 11-message exchanges.
3 switchable personas:
- Real Estate Agent 🏠
- Developer/DevOps 💻
- Healthcare/Dental 🏥

Messages reveal one by one with timed delays.
On completion: routes to guide generation with demo transcript.

### 5.4 Guide Generation — `/guide-generation` (`SetupGuideView.jsx`)

Uses `useGuideStream` hook:
1. `POST /generate-guide` → receives `guide_id`
2. `GET /events/{guide_id}` (SSE) for real-time progress
3. Fallback: poll `GET /guide/{guide_id}` every 3s

**LoadingScreen** displays:
- Current stage label (e.g., "Reading knowledge base...")
- Turn progress (N / 40 max turns)
- Elapsed time
- Quality meter (animated fill)
- Fun facts ticker (rotating tips)
- Cost tracking (usually ~$2 per guide)

Timeout: 10 minutes max. Graceful retry on failure.

### 5.5 Guide Output — rendered by `OutputDisplay.jsx`

Three tabs:
| Tab | Content |
|---|---|
| **Setup Guide** | Full markdown guide (12+ sections, 20k+ chars) |
| **Reference Docs** | Expandable accordion cards per document |
| **Prompts** | Initialization prompts to paste into OpenClaw |

**Features:**
- Sticky table of contents (desktop) / floating TOC button (mobile)
- `Scorecard` component: context depth %, sections covered, follow-up alerts
- Before/After transformation teaser
- Copy individual sections / Download all as markdown
- "Copy All Prompts" bulk extraction
- Share link generation (`/view/:guideId`)
- Retry button for failed guides

**Markdown rendering** (`MarkdownRenderer.jsx`):
- Custom h2/h3 anchors for TOC linking
- Callout classification (⚠️ warning, 💡 tip, ✅ action)
- Syntax-highlighted code blocks with language label + copy button
- Dark prose theme (`.prose-dark` overrides)
- GFM tables with alternating rows
- Styled checkboxes for task lists
- Images with rounded borders and shadows

### 5.6 Shareable Guide — `/view/:guideId` (`GuidePageView.jsx`)

Public URL for sharing generated guides.
- Polls `GET /guide/{guideId}` every 3s
- Handles states: `complete`, `generating`, `error`, `not_found`
- 10-minute timeout with graceful retry
- Uses same `OutputDisplay` component for rendering

## 6 | API Integration

All requests proxied through Vite dev server to `http://localhost:8000`.

| Method | Endpoint | Purpose |
|---|---|---|
| `GET` | `/health` | Backend readiness check |
| `POST` | `/format` | Format raw Vapi transcript |
| `POST` | `/generate-guide` | Trigger guide generation → returns `guide_id` |
| `GET` | `/events/{guideId}` | SSE stream for generation progress |
| `GET` | `/guide/{guideId}` | Fetch completed guide (polling fallback) |
| `POST` | `/retry-guide/{guideId}` | Retry failed generation |
| `GET` | `/demos` | List available demo guides |
| `GET` | `/mock-generate?demo_id=X` | Load pre-generated demo instantly |
| `GET` | `/demo-stream/{demoId}` | SSE replay of demo at 10x speed (~20s) |
| `POST` | `/webhook` | Vapi end-of-call webhook (backend-to-backend) |

**Guide data shape:**
```js
{
  guide_id: string,
  status: 'complete' | 'generating' | 'error' | 'not_found',
  outputs: {
    setup_guide: string,          // markdown
    reference_documents: [        // array of { name, content }
      { name: string, content: string }
    ],
    prompts: string,              // markdown with code blocks
    quality_check: string
  },
  scorecard: {
    context_depth: number,        // 0–1
    sections_covered: number,
    sections_total: number,
    follow_ups: string[]
  },
  transcript_excerpt: string
}
```

## 7 | State Management

No external state library. React hooks + component state only.

**App-level** (`App.jsx`):
```
phase: 'landing' | 'interview' | 'demo-interview' | 'processing' | 'mock-loading' | 'mock-output'
transcriptData, mockGuide, recoveryBanner
```

**Interview** (`useVapi.js`):
```
callStatus: 'idle' | 'connecting' | 'active' | 'ended'
voiceState: 'idle' | 'user-speaking' | 'agent-thinking' | 'agent-speaking'
transcript: Message[], formattedTranscript, error
```

**Guide generation** (`useGuideStream.js`):
```
guideData, loading, progress: { stage, turn, maxTurns, cost, tokens }
```

**Persistence** (LocalStorage):
- `easyclaw_guide_history` — completed guides (max 50)
- `easyclaw_transcript_backup` — crash recovery (24hr expiry)

## 10 | Anti-Patterns (Do Not Introduce)

- Do not add Next.js, TypeScript, or any framework migration
- Do not add a state management library (Zustand, Redux, etc.)
- Do not add shadcn/ui or any component library
- Do not add Framer Motion (CSS animations are intentional)
- Do not add light mode / theme toggle (dark-only by design)
- Do not add pricing tiers or payment flows (not part of this product)
- Do not add authentication or user accounts
- Do not use inline styles where Tailwind utilities exist
- Do not use `<div onClick>` for interactive elements — use `<button>`
- Do not add placeholder/dummy content — every demo is backed by real data
- Do not modify backend endpoints or data shapes
