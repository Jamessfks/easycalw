# Claude Code Implementation Guide: OpenClaw Concierge

This guide explains how to use the Master Documentation Suite to build the OpenClaw Concierge using Claude Code. Each document has a specific role in the project's architecture.

## 1. Project Overview

The OpenClaw Concierge is a three-model system designed to simplify the setup of OpenClaw agents through a consultative voice conversation.

*   **Model 1 (Gemini Live):** The consultative interviewer.
*   **Model 2 (ADK Agent):** Extracts structured JSON from the call.
*   **Model 3 (Gemini 2.5 Pro Backend):** Generates the final configuration files.

## 3. Implementation Workflow

### Step 3.1: Initialize the Project

Start by scaffolding the React frontend and FastAPI backend.

### Step 3.2: Configure the Input Layer (Model 1 & 2)

Use `consultative_system_prompt.md` and `schema.json` to configure the Gemini Live agent in the Google ADK. Ensure the agent is programmed to extract all fields defined in the schema.

### Step 3.3: Build the Backend Logic (Model 3)

1.  **Skill Lookup:** Implement a Python function that uses `registry.md` to map natural language use cases to verified skill slugs.
2.  **Config Generation:** Create an endpoint that takes the structured JSON from the ADK agent and uses `smart_markdown_templates.md` and `openclaw_ref.md` to generate the final ZIP package.
3.  **Security First:** Ensure the generated `README.md` and `setup.sh` strictly follow the no-API-key-storage policy as defined in the templates.

### Step 3.4: Integrate the Frontend

Build the React UI to initiate the call, display the live transcript, and provide the final download link from the backend.
