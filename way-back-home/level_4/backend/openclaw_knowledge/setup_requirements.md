# OpenClaw voice interview — required topics

Use this checklist when interviewing the user. Confirm each area before finishing.

## Identity & environment

- Operating system (macOS, Linux, Windows).
- Whether they use **OpenAI Codex (OAuth)** for OpenClaw auth vs other flows they mention.

## Channels & integrations

- Which surfaces they need (e.g. chat apps, email, local CLI assistant).
- Any **Tier 2/3** integrations (developer accounts, OAuth scopes) — note manual steps later.

## Skills & automation

- Which workflows they want (calendar, code, docs, browser automation, etc.).
- Risk tolerance: whether they want **permission / trust** tooling called out in the guide.

## Wrap-up

- Ask if anything is missing; offer a short summary of what you captured.
- When the user says they are **done** or **generate my guide**, remind them to tap **Generate setup guide** in the UI so the documentation agent can run.
