# Reference: Cron Jobs for Bakery Operations

**Context:** This document covers the two scheduled automations configured for the bakery. Use it to modify schedules, change delivery targets, test jobs manually, or add new automations in the future.

---

## Your Two Configured Jobs

| Job | Schedule | What It Does |
|---|---|---|
| Morning Bakery Briefing | `30 5 * * *` (5:30 AM daily, Pacific) | Compiles schedule + supplier status, delivers to Telegram |
| Supplier Order Review | `0 16 * * 0` (4:00 PM Sundays, Pacific) | Checks tracking sheet for items to reorder, delivers to Telegram |

Check them anytime:
```bash
openclaw cron list
```

---

## How Cron Scheduling Works

Cron expressions follow the format: `minute hour day-of-month month day-of-week`

| Expression | Meaning |
|---|---|
| `30 5 * * *` | 5:30 AM every day |
| `0 16 * * 0` | 4:00 PM every Sunday |
| `0 7 * * 1-5` | 7:00 AM Monday through Friday |
| `0 9,17 * * *` | 9:00 AM and 5:00 PM every day |

Always include `--tz "America/Los_Angeles"` for Pacific Time. If you omit the timezone, the gateway uses its system timezone, which may not be Pacific.

## Testing a Job Immediately

You do not need to wait until the scheduled time to test a cron job. Run it immediately:

```bash
openclaw cron list           # find the job ID
openclaw cron run <job-id>   # runs it now
```

Check the result:
```bash
openclaw cron runs --id <job-id> --limit 5
```

## Modifying an Existing Job

Change the schedule (e.g., shift morning briefing to 6:00 AM):
```bash
openclaw cron edit <job-id> --cron "0 6 * * *" --tz "America/Los_Angeles"
```

Change the prompt message:
```bash
openclaw cron edit <job-id> --message "Your updated prompt here."
```

Change the Telegram delivery target:
```bash
openclaw cron edit <job-id> --to "YOUR_NEW_TELEGRAM_CHAT_ID"
```

---

## Adding a New Automation (Example: Friday Prep Reminder)

If you want a Friday afternoon reminder to prep the weekend ingredient list:

```bash
openclaw cron add \
  --name "Friday Weekend Prep" \
  --cron "0 14 * * 5" \
  --tz "America/Los_Angeles" \
  --session isolated \
  --message "It is Friday afternoon. Check the supplier tracking sheet and Apple Reminders for anything needed before Monday. Compile a short weekend prep checklist: ingredients to pull from storage, items to thaw, anything to order for Monday delivery. Keep it brief." \
  --announce \
  --channel telegram \
  --to "YOUR_TELEGRAM_CHAT_ID"
```

**Vet any new automations** by running `openclaw cron list` after adding and confirming the new job appears correctly before relying on it.

## Deleting a Job

```bash
openclaw cron list            # find the job ID
openclaw cron remove <job-id>
```

Verify it is gone:
```bash
openclaw cron list
```

---

## If a Job Stops Running

1. Check the gateway is running: `openclaw gateway status`
2. Check the Mac Mini did not sleep (verify Amphetamine is active in menu bar)
3. Check for errors: `openclaw cron runs --id <job-id> --limit 10`
4. Run doctor: `openclaw doctor`
5. Manual test: `openclaw cron run <job-id>`

For recurring jobs, OpenClaw applies exponential backoff after failures (30s → 1m → 5m → 15m → 60m). If a job failed due to a temporary API issue, it will automatically resume on the next scheduled run once the API is healthy.

---

## How Jobs Deliver to Telegram

Both jobs use `--session isolated` with `--announce`. This means:

- Each run starts a fresh, clean session (no memory of previous runs)
- The output is delivered directly to your Telegram chat
- A brief summary is also posted to the main agent session

This is the correct setup for scheduled briefings — each morning's report is independent, and you get the full output in Telegram rather than having to open the dashboard.

---

## Job Store Location

Cron jobs persist at `~/.openclaw/cron/jobs.json`. Do not edit this file while the gateway is running. If you need to edit it manually, stop the gateway first:

```bash
openclaw gateway stop
# edit the file
openclaw gateway start
```

Prefer `openclaw cron edit` over manual edits whenever possible.
