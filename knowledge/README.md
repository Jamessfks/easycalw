# EasyClaw Knowledge Base

Shared team knowledge. Any agent or team member starting a new conversation should read `state.md` first.

## Files

| File | Purpose | Update when... |
|------|---------|----------------|
| `state.md` | Current snapshot of everything | Overall state changes |
| `team.md` | Who does what, who thinks what | Roles or perspectives shift |
| `customers.md` | Customer/test interaction insights | After every customer conversation |
| `decisions.md` | Decision log with context | A decision is made |
| `debates.md` | Open disagreements | A new disagreement surfaces or one resolves |
| `vision.md` | Product vision | Vision evolves |
| `roadmap.md` | Timeline and priorities | Priorities change |
| `competitive/` | Market research (3-layer analysis) | New competitor data found |

## Rules

1. **state.md is a snapshot, not a log.** Rewrite it when things change. Keep it under 2 minutes to read.
2. **decisions.md is append-only.** Never delete a decision. Add context if it gets revisited.
3. **debates.md tracks resolution.** When a debate resolves, mark it resolved and link to the decision.
4. **customers.md captures insights, not transcripts.** Extract the juice, not the full conversation.
5. **No empty templates.** If a file has nothing to say yet, don't create it.
